## 1. Trạng thái hệ thống hiện tại — CÁI GÌ ĐÃ CHẠY ĐƯỢC

### Auth (đã fix hoàn toàn trong session hôm nay)

Login flow hoạt động ổn định. Kiến trúc hiện tại (do backend chưa implement httpOnly cookie):

```
Browser → POST /auth/login (Server Action)
        → Backend trả { data: { user: { roles, username, storeId, ... } } }
        → Web tạo short-lived token (base64url, exp: 30s)
        → redirect → /api/auth-callback?t=TOKEN

/api/auth-callback GET (Route Handler):
        → decode token → validate expiry → check roles
        → NextResponse.redirect('/admin' hoặc '/store')
        → Set-Cookie: userRoles=role1|role2   (httpOnly, 30 phút)
        → Set-Cookie: userName=admintri       (non-httpOnly, 30 phút)
        → Set-Cookie: storeId=xxx             (non-httpOnly, 30 phút, chỉ set nếu có)

middleware.ts:
        → Guard /admin/* và /store/* bằng userRoles cookie
        → Không có cookie → redirect /login
```

**Cookies quan trọng:**

| Cookie | httpOnly | Nội dung | Dùng bởi |
|--------|----------|----------|----------|
| `userRoles` | ✅ | `admin\|customer` (separator `\|`) | middleware.ts guard |
| `userName` | ❌ | `admintri` | UserDisplay.tsx (document.cookie) |
| `storeId` | ❌ | ObjectId của quán | store/menu/page.tsx (server cookies()) |

**Logout:** `POST /api/logout` (không phải GET — GET bị Next.js prefetch và clear cookie).

## 2. Việc web cần làm — không phụ thuộc backend

### 2.1. Session cookie expiry — QUAN TRỌNG 🔴

**Vấn đề:** Cookie `userRoles`, `userName`, `storeId` hiện có `maxAge: 1800` (30 phút). Sau 30 phút, người dùng bị kick ra login mà không có cảnh báo.

**Yêu cầu spec:** Refresh token 30 ngày.

**Giải pháp đề xuất (chọn 1):**

**Option A (đơn giản nhất):** Tăng `maxAge` lên `86400` (24h) hoặc `2592000` (30 ngày) trong `/api/auth-callback/route.ts` lines 33–43. Phù hợp vì đây chỉ là session marker, không phải token thật.

**Option B:** Implement refresh — mỗi khi middleware cho pass, extend cookie expiry (dùng `NextResponse.next()` với `response.cookies.set()`).

**File cần sửa:** `web/app/api/auth-callback/route.ts`

```typescript
// Lines 33–43 — đổi maxAge: 1800 thành maxAge: 86400
response.cookies.set('userRoles', roles.join('|'), {
  path: '/', maxAge: 86400, sameSite: 'lax', secure, httpOnly: true,
})
```

## 4. Gotchas kỹ thuật — ĐỌC KỸ TRƯỚC KHI CHỈNH

### 4.1. KHÔNG được dùng `cookieStore.toString()` trong Server Component

Next.js 16 có bug: `cookieStore.toString()` trong Server Component trigger implicit Set-Cookie side effects, có thể clear cookies của browser.

```typescript
// ❌ SAI — sẽ clear cookies
const cookieStore = await cookies()
fetch(url, { headers: { Cookie: cookieStore.toString() } })

// ✅ ĐÚNG — đọc từng cookie cụ thể
const cookieStore = await cookies()
const token = cookieStore.get('accessToken')?.value
fetch(url, { headers: { Authorization: `Bearer ${token}` } })

// ✅ TỐT NHẤT — dùng credentials: 'include' (khi backend đã implement cookie auth)
fetch(url, { credentials: 'include', cache: 'no-store' })
```

### 4.2. KHÔNG được dùng `<Link href="/api/logout">` 

Next.js App Router tự prefetch tất cả `<Link>` visible trong viewport, kể cả API routes. `GET /api/logout` sẽ chạy và clear cookie khi page load.

```tsx
// ❌ SAI
<Link href="/api/logout">Đăng xuất</Link>

// ✅ ĐÚNG
<form action="/api/logout" method="POST">
  <button type="submit">Đăng xuất</button>
</form>
```

Logout route phải là `POST`, không phải `GET`.

### 4.3. Cookie `userRoles` dùng separator `|` (pipe), không phải `,`

Edge Runtime parser trên Vercel decode `%2C` (comma) rồi split lại theo `,`, khiến cookie bị truncate. Luôn dùng `|`:

```typescript
// Set: roles.join('|')  →  "customer|admin"
// Check: roles.includes('admin')  →  substring match, hoạt động đúng
```

### 4.4. Middleware file phải đặt tên `middleware.ts`, export `middleware`

Next.js chỉ nhận đúng 1 file tên `middleware.ts` ở project root (`web/middleware.ts`) với export `middleware`. File khác tên (như `proxy.ts`) bị bỏ qua hoàn toàn.

### 4.5. `storeId` cookie chỉ set khi login response có field `storeId`

`web/app/login/actions.ts` extract: `data?.data?.user?.storeId`. Nếu backend không trả field này, cookie không được set và `/store/menu` sẽ redirect về `/login`.

**Kiểm tra:** Đăng nhập với tài khoản store_owner → DevTools Application → Cookies → phải có `storeId`.

### 4.6. Session expire sau 30 phút (cần fix — xem mục 2.1)

Hiện `maxAge: 1800`. Người dùng sẽ bị kick ra sau 30 phút không reload.
